# Objective: specify a simple, unique identifier scheme for data that enables resource retrieval using HTTP protocol from OM cloud and local hosted sites 

Dereferenceable URI 

Two parts. Part A is opaque to URI processing and refers to the Web address of the resource.  Part A is handled by DNS and Web servers (including rewrite rules).  

Part B starts with a unique namespace, typically derived from organization Web domain name (e.g.; 'dc.gov' -> 'dcgov') with delimiting period characters removed.
